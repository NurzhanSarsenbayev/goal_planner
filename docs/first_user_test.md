# First user test script

Goal:

Validate whether the app is understandable and useful for a real first user before adding more features.

Tester:

First non-developer user.

Test type:

Manual product test on a real Android device.

## Before the test

Developer must:

- install the current debug build on the device;
- create/export one backup before the test;
- explain that the app is local-first;
- explain that backups are manual;
- explain that this is not a final polished product.

Do not explain every screen in advance. Let the tester try to understand the app.

## Test scenario

Ask the tester to complete these tasks without step-by-step help.

### 1. Create a goal

Task:

Create one real goal that matters to you.

Observe:

- Can the tester find where to create a goal?
- Is the wording clear?
- Does the goal screen feel understandable?

### 2. Add tasks to the goal

Task:

Add 2-3 tasks that would move this goal forward.

Observe:

- Can the tester understand where goal tasks live?
- Is the difference between goal and task clear?
- Does task creation feel too hidden or too heavy?

### 3. Plan something for today

Task:

Schedule one task for today.

Observe:

- Can the tester find how to schedule a task?
- Does Today make sense?
- Is it clear what is planned for today?

### 4. Complete a task

Task:

Complete one planned task.

Observe:

- Is completion obvious?
- Is feedback clear?
- Does progress feel useful?

### 5. Create a habit

Task:

Create one habit you might track for a week.

Observe:

- Can the tester find habits?
- Is the habit UI understandable?
- Does the tester understand what to do daily?

### 6. Use Calendar

Task:

Open Calendar and check what is planned.

Observe:

- Does Calendar help or confuse?
- Are scheduled tasks easy to understand?
- Does the selected day behavior make sense?

### 7. Check Reports

Task:

Open Reports and explain what you think the numbers mean.

Observe:

- Are reports understandable?
- Are habits and tasks clearly separated?
- Are the metrics useful or confusing?

### 8. Backup safety check

Task:

Find backup/export actions in More.

Observe:

- Does the tester understand why backup matters?
- Is the wording scary enough for restore?
- Is the backup area too technical?

### 9. Reminder return loop check

Task:

Find where reminder settings live and explain when you expect the app to remind you.

Observe:

- Can the tester find reminders without help?
- Does the tester understand the difference between standalone reminders and Daily Review Reminder?
- Is the Daily Review Reminder useful or annoying?
- Does the tester expect task reminders, habit reminders or a general daily reminder first?

### 10. Habit reminder check

Task:

Create a habit, enable a reminder for it, choose a time, then mark the habit as done/skipped and explain what you expect from the reminder.

Observe:

- Can the tester find the reminder option inside the habit dialog?
- Does the reminder control feel too hidden or too heavy?
- Does the tester understand that done/skipped/failed suppress today's reminder?
- Does the tester expect habit reminders to be separate from standalone reminders?
- Does the tester understand that changing app language updates future reminder notifications?

## Questions after the test

Ask:

1. What did you expect this app to help you with?
2. What was clear?
3. What was confusing?
4. What felt unnecessary?
5. What did you want to do but could not?
6. Would you use this tomorrow?
7. What is the one thing that must improve first?

## Developer notes

During the test:

- do not defend the app;
- do not explain unless the tester is completely blocked;
- write down exact confusion points;
- write down exact words the tester uses;
- separate product issues from visual polish issues.

## Result format

After the test, summarize findings as:

### Must fix before next test

- ...

### Should fix soon

- ...

### Later

- ...

### Product insight

- ...
